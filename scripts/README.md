# OpenPCB Module CLI

Unified command-line interface for all module operations.

## Quick Start

### Interactive Menu

```bash
npm run module
```

Shows an interactive menu with all available commands:

- Create new module
- Validate modules
- Run all codegen
- Generate module registry
- Generate Rust types
- Generate bridge bindings
- Generate module SDKs

### Direct Commands

```bash
# Create new module
npm run module:create

# Validate all modules
npm run module:validate

# Run full codegen pipeline
npm run module:codegen

# Or run the CLI directly
bun scripts/module-cli.ts
bun scripts/module-cli.ts --create
bun scripts/module-cli.ts --codegen
```

## Architecture

```
scripts/
├── module-cli.ts              ← Main CLI entrypoint
├── lib/
│   ├── commands/              ← Command implementations
│   │   ├── create.ts          ← Module creation
│   │   ├── generate-registry.ts    ← Module registry
│   │   ├── generate-sdk.ts         ← SDK generation
│   │   ├── generate-types.ts       ← Rust→TS types
│   │   └── generate-bridge.ts      ← Bridge bindings
│   ├── types.ts               ← Shared types
│   ├── utils.ts               ← Utilities
│   ├── validation.ts          ← Validation logic
│   ├── templates.ts           ← Code templates
│   ├── prompts.ts             ← Interactive prompts
│   └── rollback.ts            ← Error recovery
├── gen-modules.ts             ← Legacy (called by generate-registry)
├── gen-sdk.ts                 ← Legacy (called by generate-sdk)
├── gen-types.ts               ← Legacy (called by generate-types)
└── gen-bridge.ts              ← Legacy (called by generate-bridge)
```

## Commands

### Create Module

```bash
npm run module:create
```

Interactive module scaffolding:

- Prompts for ID, label, kind, namespace
- Smart defaults based on module kind
- Optional HTTP endpoints, Rust commands, React UI
- Automatic validation
- Auto-registration in Cargo workspace
- Optional codegen after creation

### Validate Modules

```bash
npm run module:validate
```

Validates all module manifests:

- JSON schema validation
- Namespace/ID uniqueness
- Dependency cycle detection
- Service export conflicts
- File existence checks

### Generate Code

```bash
npm run module:codegen        # Full pipeline
npm run modules:generate      # Registry only
npm run types:generate        # Rust types only
npm run bridge:generate       # Bridge bindings only
npm run sdk:generate          # SDKs only
```

**Full codegen pipeline:**

1. Module registry (`src/core/frontend/src/generated/modules.ts`)
2. Rust types (`src/core/frontend/src/generated/types/`)
3. Bridge bindings (`src/core/frontend/src/bridge/generated/`)
4. Module SDKs (`src/core/frontend/src/generated/sdk/`)

## CLI Flags

```bash
--create              Create new module
--validate            Validate manifests
--codegen             Run full codegen
--codegen-registry    Registry only
--codegen-types       Types only
--codegen-bridge      Bridge only
--codegen-sdk         SDK only
--help, -h            Show help
```

## Module Creation Workflow

1. **Run CLI**

   ```bash
   npm run module:create
   ```

2. **Answer prompts**
   - Module ID (e.g., `task-manager`)
   - Label (e.g., `Task Manager`)
   - Kind (space/service/widget/integration/system)
   - HTTP endpoints? (yes/no)
   - Rust commands? (yes/no)
   - React UI? (yes/no, auto-yes for space)
   - Namespace (smart default)
   - Version (default: 0.1.0)
   - Tags (optional)
   - Run codegen? (default: yes)

3. **Review configuration**
   - CLI shows summary
   - Confirm or cancel

4. **Module created**
   - Files scaffolded
   - Cargo workspace updated (if Rust)
   - Codegen runs (if requested)
   - Success summary with next steps

## Generated Files

### Always Created

- `modules/{id}/manifest.json` - Module manifest
- `modules/{id}/ts/module.ts` - TypeScript entry

### Optional (based on options)

- `modules/{id}/react/Space.tsx` - React component
- `modules/{id}/rust/Cargo.toml` - Cargo manifest
- `modules/{id}/rust/src/lib.rs` - Rust library
- `modules/{id}/rust/src/bin/export_types.rs` - Type export

### Generated by Codegen

- `src/core/frontend/src/generated/modules.ts` - Module registry
- `src/core/frontend/src/generated/types/module-{id}.ts` - Rust types
- `src/core/frontend/src/generated/sdk/{id}.ts` - Module SDK
- `src/core/frontend/src/bridge/generated/` - Bridge bindings

## Development

### Adding New Commands

1. Create command file in `scripts/lib/commands/`
2. Export async function
3. Add to `module-cli.ts`:

   ```typescript
   import { myCommand } from "./lib/commands/my-command.js";

   // Add to COMMANDS array
   // Add to executeCommand switch
   ```

### Modifying Templates

Edit `scripts/lib/templates.ts`:

- `generateManifest()` - manifest.json
- `generateReactComponent()` - React component
- `generateModuleEntry()` - TypeScript entry
- `generateRustLib()` - Rust library
- `generateRustCargoToml()` - Cargo.toml
- `generateRustExportTypes()` - Type export binary

### Validation Rules

Edit `scripts/lib/validation.ts`:

- Module ID format
- Namespace format
- Manifest schema
- Dependency conflicts
- File existence

## Troubleshooting

### "Module already exists"

Choose different ID or delete existing module.

### "Namespace conflict"

Choose different namespace or check existing modules.

### "Cargo.toml not found"

Ensure you're in repository root. Rust workspace required for Rust modules.

### "Bun version too old"

Upgrade Bun: `curl -fsSL https://bun.sh/install | bash`
Required: Bun >= 1.3.0

### Codegen fails

1. Check module manifests are valid: `npm run module:validate`
2. Ensure Rust crates compile: `cargo check`
3. Build bridge-introspect: `cargo build -p bridge-introspect`
